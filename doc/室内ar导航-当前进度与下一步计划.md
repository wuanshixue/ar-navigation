# 室内 AR 导航当前进度与下一步计划

## 当前结论

项目方向已经明确切换为“室内锚点 + 预设路线”的方案，不再以 GPS 作为室内主定位。

当前已经完成从起点选择、目的地选择到路线加载和场景标识展示的基础闭环，具备继续做产品化验证的条件。

## 已完成内容

- 完成室内锚点与室内路线的数据结构整理
- 完成起点锚点选择和目的地选择 UI
- 完成按起点与目的地加载对应室内路线
- 完成室内锚点、科室标点和文本标签展示
- 完成调试面板、折叠展开和基础状态查看
- 完成多条从前台到各科室的模拟路线配置

## 当前存在的问题

- 标点和文字的可读性还可以继续提升
- 当前目标与其他目标的视觉区分还不够强
- 路线与锚点仍以手工配置为主，后续维护成本会增加
- 室内校准能力还未形成稳定闭环

## 下一步计划

### 1. 优化可视化表现

- 让标签更易读
- 强化当前目标的视觉提示
- 降低其他标点对视线的干扰

### 2. 补齐室内校准能力

- 基于起点锚点做室内校准
- 提高路线在真实空间中的稳定性

### 3. 继续整理配置结构

- 提升路线和目的地配置的可维护性
- 为后续多楼层、多区域扩展做准备

### 4. 准备业务化验证

- 结合真实就诊场景验证导航流程
- 从技术验证逐步过渡到业务验证

## 汇报口径

现阶段项目已完成室内导航基础链路搭建，下一步重点不是继续堆路线，而是提升可视化效果、补齐室内校准，并为后续业务化落地做准备。
